Section 2: Agent Creation Flow

How a lazy prompt becomes a running agent — the progressive expansion pipeline

The User Journey

Step 1 — User drops a lazy prompt
You: scan my emails every morning and send me a summary on whatsapp
Step 2 — Orchestrator asks 1-2 critical questions
Claw: I'll set this up for you. Quick questions:

1. Which email? (Gmail, Outlook, or other?)
2. What time should the summary arrive?
You: gmail, 8am
Step 3 — Orchestrator generates expanded spec with smart defaults
Claw: Here's what I'll build:

Agent: email-morning-summary
Schedule: Daily at 7:45 AM (15min buffer for processing)
Integrations: Gmail API (read), WhatsApp Business API (send)
Behavior:
  - Fetch unread emails from last 24h
  - Categorize: urgent, actionable, FYI, newsletters
  - Summarize each category (max 2-3 lines per email)
  - Send via WhatsApp as a formatted message
  - Mark urgent items with priority flag
Error handling: Retry 3x, then notify via WhatsApp
Credentials needed: GMAIL_OAUTH, WHATSAPP_API_KEY

Want to tweak anything, or should I build it?
Step 4 — Build, validate, deploy
You: build it
✓ Generated agent definition
✓ Created Gmail integration worker
✓ Created WhatsApp sender worker
✓ Spec validation passed
✓ Code validation passed
⚠ Missing credential: GMAIL_OAUTH
⚠ Missing credential: WHATSAPP_API_KEY

Agent created at ~/.autopilot/agents/email-morning-summary/
Set up credentials to activate:
  agentspan credentials set GMAIL_OAUTH
  agentspan credentials set WHATSAPP_API_KEY
Step 5 — Agent runs, chat thread becomes control plane
[ Next morning, 7:45 AM — agent fires ]
Claw: Your email summary ran successfully. Sent to WhatsApp at 7:58 AM.
Processed: 47 emails
Urgent: 2 items flagged
You: also include my calendar for the day
Claw: Got it. I'll add Google Calendar integration.
⚠ Need GOOGLE_CALENDAR_OAUTH credential.
Updated agent — next run will include today's schedule.

Pipeline Under the Hood

User prompt → Orchestrator expands → Spec guardrail validates → Orchestrator generates code → Code guardrail validates → Integration check → Deploy guardrail (dry-run compile) → Deploy & schedule

Each guardrail uses Agentspan's native guardrail system. On failure, the orchestrator gets the error and self-corrects (up to 3 retries) before surfacing to the user.
